# 结构化提示词

> **模块目标**: 学会使用序号方式和JSON方式编写结构化提示词，提升AI输出的准确性和稳定性

---

## 一、为什么需要结构化提示词

### 1.1 自然语言的局限性

在日常对话中，我们习惯用自然语言表达需求。但对于AI来说，自然语言存在以下问题：

| 问题 | 示例 |
|-----|------|
| **歧义性** | "写一个简短的介绍" - 多短算简短？ |
| **信息混杂** | 背景信息、任务要求、输出格式混在一起 |
| **优先级不清** | AI不知道哪些要求是必须的，哪些是可选的 |
| **遗漏风险** | 复杂需求容易遗漏关键信息 |

### 1.2 结构化提示词的优势

**结构化提示词**就是用清晰的格式组织你的需求，让AI更容易理解和执行。

```
❌ 非结构化提示词：
帮我写一个咖啡店推广文案，要小红书风格，突出手冲咖啡特色，200字左右，目标用户是上班族，要包含地址和营业时间

✅ 结构化提示词：
## 角色
你是一位小红书博主，擅长写探店文案

## 背景
- 店铺：XX咖啡店
- 位置：写字楼聚集区
- 特色：手冲咖啡、安静环境

## 任务
写一篇推广文案

## 要求
1. 风格：小红书风格，轻松活泼
2. 篇幅：200字左右
3. 目标用户：上班族
4. 必须包含：店铺地址、营业时间
```

### 1.3 结构化提示词的核心价值

| 价值      | 说明            |
| ------- | ------------- |
| **准确性** | AI能更精准地理解每个要素 |
| **稳定性** | 多次执行结果更一致     |
| **可复用** | 模板化后可反复使用     |
| **可调试** | 问题定位更容易       |

## 二、序号方式书写

### 2.1 基本格式

序号方式是最简单直观的结构化方法，使用Markdown标题和数字列表组织内容。

#### 标准模板

```markdown
## 角色
[定义AI扮演的角色]

## 背景
[提供任务背景信息]
- 背景点1
- 背景点2

## 任务
[明确要完成的具体任务]

## 要求
1. [要求1]
2. [要求2]
3. [要求3]

## 输出格式
[描述期望的输出形式]
```

### 2.2 实战案例

#### 案例1：撰写产品介绍

```markdown
## 角色
你是一位有5年经验的电商文案策划，擅长撰写吸引人的产品介绍

## 背景
- 产品：智能保温杯
- 品牌：温享
- 核心卖点：72小时保温、LED温度显示、304不锈钢
- 目标用户：25-40岁上班族
- 价格区间：199-299元

## 任务
为这款保温杯撰写电商详情页的产品介绍文案

## 要求
1. 突出3个核心卖点
2. 语气亲切，像朋友推荐
3. 包含使用场景描述
4. 字数控制在300字以内
5. 结尾包含购买引导

## 输出格式
直接输出文案内容，不需要解释
```

#### 案例2：会议纪要整理

```markdown
## 角色
你是一位专业的行政助理，擅长整理会议纪要

## 背景
以下是我部门周会的录音转文字内容：
[粘贴会议内容]

## 任务
将以上内容整理为规范的会议纪要

## 要求
1. 提取核心讨论议题（不超过5个）
2. 记录每个议题的结论或下一步行动
3. 明确责任人和截止时间
4. 使用简洁的商务语言

## 输出格式
请按以下格式输出：

### 会议主题：[主题]
### 会议时间：[时间]
### 参会人员：[人员]

### 讨论议题
1. [议题1]
   - 讨论要点：...
   - 结论：...

### 待办事项
| 序号 | 任务 | 负责人 | 截止日期 |
|-----|------|-------|---------|
```

### 2.3 序号方式的技巧

#### 技巧1：使用层级结构

```markdown
## 要求
1. 内容要求
   1.1 突出产品核心卖点
   1.2 使用场景化描述
2. 格式要求
   2.1 段落分明
   2.2 适当使用emoji
3. 字数要求
   3.1 总字数：300-500字
```

#### 技巧2：明确优先级

```markdown
## 要求
【必须】
- 必须包含产品价格
- 必须包含购买链接

【建议】
- 可以加入用户评价
- 可以加入使用小贴士

【禁止】
- 不要使用夸张宣传语
- 不要出现竞品对比
```

#### 技巧3：提供示例

```markdown
## 输出格式示例
请参考以下格式输出：

---
**产品名称**：XX咖啡店招牌拿铁

**推荐指数**：⭐⭐⭐⭐⭐

**体验感受**：
[100字左右的体验描述]

**适合人群**：上班族、咖啡爱好者

**人均消费**：35-50元
---
```

## 三、JSON方式书写

### 3.1 什么是JSON格式

JSON（JavaScript Object Notation）是一种结构化数据格式，用键值对的方式组织信息。

**为什么用JSON写提示词？**
- 结构更加严谨
- 适合程序化处理
- 复杂参数更清晰
- 便于API调用场景

### 3.2 基本格式

```json
{
  "role": "角色定义",
  "background": {
    "key1": "背景信息1",
    "key2": "背景信息2"
  },
  "task": "具体任务",
  "requirements": [
    "要求1",
    "要求2"
  ],
  "output_format": "输出格式说明"
}
```

### 3.3 实战案例

#### 案例1：内容生成任务

```json
{
  "role": "资深新媒体运营",
  "background": {
    "platform": "小红书",
    "account_type": "美食探店博主",
    "fans_count": "10万+",
    "content_style": "轻松活泼、真实分享"
  },
  "task": "为一家新开的日式拉面店写探店文案",
  "store_info": {
    "name": "一兰拉面",
    "location": "市中心购物广场B1",
    "specialty": "豚骨拉面",
    "price_range": "人均60-80元",
    "highlights": ["24小时营业", "单人隔间设计", "可自定义面条硬度"]
  },
  "requirements": [
    "标题吸引眼球，使用emoji",
    "正文200-300字",
    "突出店铺特色",
    "包含实用信息（地址、营业时间）",
    "语气像分享给闺蜜"
  ],
  "output_format": {
    "title": "吸引人的标题",
    "content": "正文内容",
    "tags": ["标签1", "标签2", "标签3"]
  }
}
```

#### 案例2：数据分析任务

```json
{
  "role": "数据分析师",
  "background": {
    "company": "某电商平台",
    "data_type": "用户购买行为数据",
    "time_range": "2024年Q1"
  },
  "task": "分析用户购买数据，输出分析报告",
  "data": {
    "total_orders": 15000,
    "total_revenue": 2500000,
    "new_users": 3000,
    "repeat_rate": "35%"
  },
  "analysis_dimensions": [
    "月度销售趋势",
    "用户复购分析",
    "品类销售占比"
  ],
  "requirements": [
    "每个维度给出2-3个核心发现",
    "使用数据支撑结论",
    "给出可行的优化建议"
  ],
  "output_format": {
    "summary": "100字以内的总体概述",
    "analysis": [
      {
        "dimension": "维度名称",
        "findings": ["发现1", "发现2"],
        "suggestion": "优化建议"
      }
    ],
    "conclusion": "总结与下一步行动"
  }
}
```

### 3.4 JSON方式的技巧

#### 技巧1：嵌套结构处理复杂信息

```json
{
  "task": "生成营销邮件",
  "target_audience": {
    "demographics": {
      "age_range": "25-40",
      "occupation": "白领",
      "income_level": "中等偏上"
    },
    "pain_points": [
      "工作压力大",
      "缺乏运动时间"
    ],
    "preferences": {
      "tone": "专业但亲切",
      "reading_time": "2-3分钟"
    }
  }
}
```

#### 技巧2：使用数组表示列表

```json
{
  "requirements": [
    {"priority": "high", "item": "必须包含产品价格"},
    {"priority": "medium", "item": "建议加入用户评价"},
    {"priority": "low", "item": "可以加入使用教程"}
  ]
}
```

#### 技巧3：预定义输出结构

```json
{
  "output_format": {
    "type": "object",
    "properties": {
      "title": {"type": "string", "maxLength": 50},
      "content": {"type": "string"},
      "keywords": {"type": "array", "maxItems": 5}
    }
  }
}
```

## 四、两种方式对比

### 4.1 对比表格

| 维度       | 序号方式      | JSON方式       |
| -------- | --------- | ------------ |
| **学习难度** | 低，易于上手    | 中，需要了解JSON语法 |
| **可读性**  | 高，人类友好    | 中，结构清晰但不够直观  |
| **适用场景** | 日常对话、简单任务 | 复杂任务、API调用   |
| **调试难度** | 低         | 中            |
| **程序处理** | 困难        | 容易           |
| **模板复用** | 需要手动调整    | 可程序化填充       |

### 4.2 选择建议

**推荐使用序号方式的场景：**
- ✅ 日常使用ChatGPT、Claude等对话工具
- ✅ 任务相对简单，参数不多
- ✅ 需要快速迭代调整
- ✅ 团队成员技术水平不一

**推荐使用JSON方式的场景：**
- ✅ 通过API调用AI
- ✅ 需要程序化生成提示词
- ✅ 任务复杂，参数众多
- ✅ 需要严格的结构化输出

## 五、最佳实践

### 5.1 通用原则

无论使用哪种方式，都应遵循以下原则：

1. **信息完整**：角色、背景、任务、要求缺一不可
2. **结构清晰**：使用标题、列表组织内容
3. **要求具体**：避免模糊表述，给出明确指标
4. **示例辅助**：复杂格式提供参考示例

### 5.2 常见错误

#### 错误1：结构混乱

```markdown
❌ 错误示例：
你是一个文案，帮我写个咖啡店推广，要小红书风格，对了还有包含地址，大概200字吧，受众是上班族

✅ 正确示例：
## 角色
小红书博主，擅长探店文案

## 任务
为咖啡店撰写推广文案

## 要求
1. 风格：小红书风格
2. 篇幅：200字左右
3. 受众：上班族
4. 必须包含：店铺地址
```

#### 错误2：要求冲突

```markdown
❌ 错误示例：
## 要求
1. 详细介绍产品所有功能
2. 字数控制在100字以内

✅ 正确示例：
## 要求
1. 介绍产品3个核心功能
2. 每个功能用1-2句话概括
3. 总字数控制在150字以内
```

### 5.3 调试技巧

当AI输出不理想时，按以下步骤调试：

1. **检查结构**：每个模块是否清晰？
2. **检查要求**：是否有歧义或冲突？
3. **添加示例**：是否需要提供参考？
4. **简化任务**：是否一次要求太多？
