# 需求拆分技巧

> **模块目标**: 掌握需求拆分的核心方法，能将复杂任务拆解为AI可执行的指令

---

## 一、为什么需要拆分需求

### 1.1 AI理解能力的边界

AI虽然"读过"海量文字，但它不会读心术。

**一个常见的问题场景：**

```
❌ 用户说：帮我写个文案

🤖 AI内心：写什么文案？给谁看？什么风格？多长？用在哪儿？
```

结果：AI只能"猜"你的需求，输出往往不尽如人意。

### 1.2 好问题 = 好答案

AI的输出质量，很大程度上取决于你输入的质量。

| 输入质量 | AI输出质量 |
|---------|-----------|
| 模糊的需求 | 模糊的答案 |
| 清晰的需求 | 精准的答案 |
| 结构化的需求 | 专业级的答案 |

**记住这个公式：**

> **你的提示词质量 = AI输出质量的天花板**

### 1.3 复杂任务 vs 简单任务

| 任务类型 | 特点 | AI表现 |
|---------|------|-------|
| 简单任务 | 单一目标、明确格式 | 一次搞定 |
| 复杂任务 | 多个目标、需要推理 | 需要拆解 |

**举个例子：**

```
简单任务：把这段话翻译成英文
复杂任务：帮我做一份市场分析报告

↓ 复杂任务需要拆解 ↓

1. 确定分析的目标市场
2. 明确分析维度（竞品、用户、趋势等）
3. 指定报告格式和篇幅
4. 分步骤生成各部分内容
```

## 二、需求拆分的核心方法

本章介绍四种经过验证的需求拆分方法，它们可以单独使用，也可以组合使用。

### 2.1 方法一：角色设定法

**核心原理**：给AI一个身份角色，让它以特定专业视角来回答问题。

#### 为什么角色设定有效？

```
没有角色设定：
用户：怎么写周报？
AI：写周报要包含工作内容、完成情况、下周计划...（泛泛而谈）

有角色设定：
用户：你是一位有10年经验的互联网大厂管理者，请告诉我怎么写周报
AI：作为管理者，我认为一份好的周报应该...（专业、有深度）
```

#### 角色设定的三要素

| 要素 | 说明 | 示例 |
|-----|------|------|
| 身份 | AI扮演什么角色 | 资深文案、数据分析师、产品经理 |
| 背景 | 角色的专业背景 | 10年经验、专注电商领域、擅长B端产品 |
| 目标 | 这个角色要帮你做什么 | 帮我优化文案、分析数据、设计产品方案 |

#### 实战案例

**案例1：写求职简历**

```
❌ 普通提示词：
帮我写一份简历

✅ 角色设定提示词：
你是一位有15年HR经验的资深招聘专家，熟悉互联网行业的人才评估标准。
请帮我优化以下简历内容，使其更符合产品经理岗位的要求：
[粘贴简历内容]
```

**案例2：学习新知识**

```
❌ 普通提示词：
解释一下什么是机器学习

✅ 角色设定提示词：
你是一位擅长用通俗语言讲解技术概念的科普作家，读者是没有技术背景的普通人。
请用生活中的例子解释什么是机器学习，要求：
1. 不使用专业术语
2. 至少举2个生活中的类比
3. 篇幅控制在300字以内
```

### 2.2 方法二：背景信息法

**核心原理**：提供充足的上下文信息，让AI理解任务的全貌。

#### 背景信息的四个维度

| 维度 | 问题 | 为什么重要 |
|-----|------|-----------|
| 任务背景 | 为什么要做这件事？ | AI理解目的后能更好地把握方向 |
| 目标受众 | 给谁看/谁使用？ | 不同受众需要不同的表达方式 |
| 使用场景 | 在哪里用？ | 场景决定内容的风格和格式 |
| 特殊要求 | 有什么限制？ | 避免AI输出不符合要求的内容 |

#### 实战案例

**案例1：写营销文案**

```
❌ 缺少背景信息：
帮我写一个咖啡店的推广文案

✅ 完整背景信息：
【背景信息】
- 任务：为一新开业的精品咖啡店写推广文案
- 受众：25-35岁的都市白领，注重生活品质
- 场景：发布在小红书平台
- 特点：这家店主打手冲咖啡，装修风格是日式简约风
- 目标：吸引顾客到店打卡

请根据以上信息，写一篇小红书风格的推广文案，要求：
1. 标题吸引眼球
2. 内容包含店铺特色
3. 语气轻松活泼
4. 字数200-300字
```

**案例2：写工作邮件**

```
❌ 缺少背景信息：
帮我写一封申请延期的邮件

✅ 完整背景信息：
【背景信息】
- 我是谁：某互联网公司的产品经理
- 收件人：项目组领导和客户方负责人
- 项目情况：App开发项目，原定3月15日上线
- 延期原因：第三方接口对接遇到技术问题，需要额外一周解决
- 新的交付时间：3月22日
- 语气要求：诚恳、专业，但不卑不亢

请帮我写一封邮件，说明延期情况并申请延期。
```

### 2.3 方法三：步骤拆解法

**核心原理**：将复杂任务分解为多个简单步骤，逐步引导AI完成。

#### 什么时候需要步骤拆解？

- ✅ 任务包含多个子任务
- ✅ 需要AI进行推理或规划
- ✅ 输出内容较长或结构复杂
- ✅ 一次生成的效果不理想

#### 拆解原则

```
复杂任务 → 拆分成 → 多个简单子任务 → 逐个完成 → 整合结果
```

#### 实战案例

**案例1：写一篇完整文章**

```
❌ 一次性要求：
帮我写一篇关于时间管理的文章，3000字

✅ 步骤拆解：

第一步：生成大纲
我需要写一篇关于时间管理的文章，目标读者是职场新人。
请先帮我生成一个文章大纲，包含：
1. 吸引人的标题（3个备选）
2. 文章结构（至少4个章节）
3. 每个章节的核心观点

第二步：逐章节撰写
（等AI生成大纲后）
很好！现在请帮我写第一章的内容，约500字...

第三步：优化润色
（完成初稿后）
请帮我优化这篇文章的开头，让它更有吸引力...
```

**案例2：数据分析任务**

```
❌ 一次性要求：
分析这份销售数据，给我一个报告

✅ 步骤拆解：

第一步：数据理解
以下是我公司2024年Q1的销售数据：
[粘贴数据]
请先帮我理解这份数据的结构和特点。

第二步：维度分析
请从以下维度分析这份数据：
1. 月度销售趋势
2. 产品类别占比
3. Top 10 畅销产品

第三步：结论建议
基于以上分析，请给出：
1. 3个核心发现
2. 2条改进建议
```

### 2.4 方法四：示例引导法

**核心原理**：给AI看你期望的输出格式或风格，让它"照葫芦画瓢"。

#### 示例引导的三种方式

| 方式 | 适用场景 | 示例 |
|-----|---------|------|
| 格式示例 | 需要特定输出格式 | 表格、列表、JSON |
| 风格示例 | 需要特定写作风格 | 小红书风、公文风、幽默风 |
| 内容示例 | 需要类似的内容 | 参考文章、竞品文案 |

#### 实战案例

**案例1：格式示例**

```
请按照以下格式整理会议纪要：

【示例格式】
---
会议主题：[主题]
会议时间：[时间]
参会人员：[人员]

讨论事项：
1. [事项1]
   - 讨论要点：...
   - 决议：...

待办事项：
| 序号 | 任务 | 负责人 | 截止日期 |
|-----|------|-------|---------|
| 1 | ... | ... | ... |
---

现在请根据以下会议内容，按上述格式整理：
[粘贴会议内容]
```

**案例2：风格示例**

```
请模仿以下小红书文案的风格，为我的民宿写一篇推广文案：

【参考文案】
"救命！这家民宿也太好住了吧😭
📍藏在山里的神仙小院，推开窗就是云海
🛏️ 房间超大的落地窗，躺看日出绝了
☕ 老板手冲咖啡一绝，早餐也很精致
📸 随便拍都是大片，原图直出！
💰人均才300+，性价比绝绝子
👉周末放松必冲！"

现在请为我的海边民宿写一篇类似风格的文案：
[民宿信息]
```

## 三、通用提示词模板

### 3.1 基础模板：BRTR公式

将以上四种方法整合，我们得到一个通用的提示词公式：

```
**BRTR公式**

B - Background（背景）：任务的背景信息
R - Role（角色）：AI扮演的角色
T - Task（任务）：具体要做什么
R - Requirement（要求）：输出的格式、风格、限制
```

#### 基础模板示例

```
【背景】我是一家新开咖啡店的店主，店铺位于写字楼附近，主要客群是上班族

【角色】你是一位有5年经验的新媒体运营专家，擅长小红书平台的内容创作

【任务】请帮我写一篇咖啡店推广文案

【要求】
- 风格：小红书风格，轻松活泼
- 篇幅：200-300字
- 内容：突出店铺特色（手冲咖啡、安静环境）
- 必须包含：店铺地址、营业时间
```

### 3.2 进阶模板：BRTRISE公式

对于更复杂的任务，可以使用进阶版：

```
**BRTRISE公式**

B - Background（背景）：任务的背景信息
R - Role（角色）：AI扮演的角色
T - Task（任务）：具体要做什么
R - Requirement（要求）：输出的格式、风格、限制
I - Input（输入）：需要AI处理的原始材料
S - Steps（步骤）：分步骤执行的说明
E - Example（示例）：期望输出的参考示例
```

#### 进阶模板示例

```
【背景】
我需要向领导汇报Q1项目进度，申请增加预算

【角色】
你是一位有10年经验的项目经理，擅长撰写专业的项目汇报

【任务】
帮我写一份项目进度汇报邮件

【要求】
- 语气：专业、简洁
- 篇幅：500字以内
- 结构：进度概述 → 成果展示 → 问题说明 → 预算申请

【输入材料】
- 项目名称：客户管理系统升级
- 当前进度：完成75%
- 主要成果：完成了用户模块、订单模块开发
- 遇到问题：第三方支付接口费用超出预期
- 申请金额：额外申请5万元预算

【步骤】
1. 先写邮件标题
2. 再写正文内容
3. 最后检查语气是否得体
```

### 3.3 模板使用建议

| 场景 | 推荐模板 | 说明 |
|-----|---------|------|
| 简单任务 | 基础模板 | 写文案、翻译、总结等 |
| 复杂任务 | 进阶模板 | 写报告、分析数据、策划方案 |
| 第一次尝试 | 基础模板 | 先简单尝试，效果不好再加信息 |
| 效果不理想 | 进阶模板 | 添加更多背景、示例、步骤 |

## 四、常见错误与避坑指南

### 4.1 错误一：指令太模糊

```
❌ 错误示例：
帮我写个东西

✅ 正确示例：
帮我写一封求职邮件，应聘产品经理岗位，收件人是字节跳动的HR
```

**避坑建议**：永远不要让AI"猜"你的需求，把"写什么、给谁看、什么风格"说清楚。

### 4.2 错误二：一次要求太多

```
❌ 错误示例：
帮我分析这份数据，写一份报告，再做PPT大纲，顺便写个演讲稿

✅ 正确示例：
（分步骤）
1. 先帮我分析这份数据的核心发现
2. （等第一步完成）基于分析结果，写一份报告大纲
3. （等第二步完成）根据大纲，帮我写PPT结构
```

**避坑建议**：复杂任务要拆解，一次只让AI做一件事。

### 4.3 错误三：没有提供背景信息

```
❌ 错误示例：
帮我写一个产品介绍

✅ 正确示例：
帮我为一款智能手表写产品介绍：
- 目标用户：25-40岁的健身爱好者
- 核心卖点：7天续航、50米防水、专业运动模式
- 使用场景：电商详情页
- 竞品：Apple Watch、华为手表
```

**避坑建议**：提供的信息越多，AI的输出越精准。

### 4.4 错误四：期望一次就完美

```
❌ 错误心态：
AI写的第一版就是最终版，不满意就是AI不行

✅ 正确心态：
AI的输出是"初稿"，需要通过对话迭代优化

迭代示例：
第一版输出 → "太长了，精简到300字"
第二版输出 → "语气太正式，改得活泼一点"
第三版输出 → "很好，再给我两个备选标题"
```

**避坑建议**：把AI当成可以不断修改的"实习生"，通过多轮对话优化结果。

## 五、实战演练

### 演练1：优化模糊需求

**原始需求**：帮我写个年终总结

**请用BRTR公式优化**：

```
【背景】
我是一家互联网公司的运营专员，2024年主要负责用户增长项目

【角色】
你是一位擅长写工作汇报的职场导师

【任务】
帮我写一份年终总结

【要求】
- 篇幅：800-1000字
- 结构：工作成果 → 数据亮点 → 不足反思 → 明年计划
- 语气：专业但不张扬
- 必须包含：3个核心数据指标
```

### 演练2：拆解复杂任务

**任务**：帮我策划一个产品发布会

**拆解步骤**：

```
第一步：明确活动框架
- 活动主题
- 目标受众
- 核心目标

第二步：设计活动流程
- 环节安排
- 时间分配
- 人员分工

第三步：制定执行计划
- 前期准备清单
- 当天流程表
- 应急预案

第四步：撰写具体内容
- 主持人串词
- 新闻通稿
- 社交媒体文案
```

## 六、本章小结

### 核心要点回顾

1. **需求拆分的价值**：好问题=好答案，你的提示词质量决定AI输出质量
2. **四种核心方法**：
   - 角色设定法：给AI一个身份
   - 背景信息法：提供充足上下文
   - 步骤拆解法：复杂任务分步完成
   - 示例引导法：给AI看期望的输出
3. **通用模板**：BRTR公式（背景-角色-任务-要求）
4. **常见错误**：模糊、贪多、缺背景、求完美
