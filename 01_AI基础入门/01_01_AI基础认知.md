## 一、AI是什么——建立正确认知

### 1.1 AI的本质：超级助手，不是万能神

很多人第一次接触AI时，容易走向两个极端：要么觉得AI无所不能，要么觉得AI高深莫测、难以驾驭。

**其实，AI就是一个超级助手。**

想象一下，你雇佣了一个：
- 📚 读过互联网上几乎所有公开文字的助手
- ⚡ 能在几秒内整理、归纳、生成内容的助手
- 🔄 永远不知疲倦、随时待命的助手

但这个助手也有明显的"性格特点"：
- ❌ 不会读心术——你需要明确告诉它要做什么
- ❌ 可能会"编造"——会根据语义错误的进行内容拼接
- ❌ 没有真实世界经验——本质上不会创新，只是究极 **“缝合怪”**


### 1.2 大语言模型（LLM）通俗理解

现在我们使用的、你现在听说的ChatGPT、Claude、DeepSeek、豆包等等，本质上都是**大语言模型（Large Language Model，简称LLM）**。

#### 用最简单的方式理解LLM

**LLM就像一个"超级接龙高手"**。他学遍了所有的人类文本，是现在绝大多数聊天AI的核心。

但是AI的`学习`和我们人类的学习是完全不同的概念：

人类学习是了解概念，AI学习是统计学：
在AI进行训练的过程中，计算机发现`计算机`这个词汇经常出现，同时他周边经常会连接`系统` 、`硬件`、`软件`这些词汇，那么在LLM中这些内容将被关联起来。

但是LLM自己理解了这些意思吗？显然是根本没有的，他仅仅知道这些词应该在一起出现在`上下文`中。

而了解LLM则是我们打开AI的钥匙：
 - 为什么模型会出现幻觉？
 - 如何让AI能够生成跟准确的回答
 - 如何合理的给AI分配工作
 - ......

这些看起来学术的内容，其实解决的核心都源自于LLM的特点。而我们则是利用LLM的特性来进行更好的AI调度。

### 1.3 AI的能力边界：能做什么 vs 不能做什么

在我们对LLM有了一定认识之后，其实就可以清晰的了解到AI的能力边界——**他是一个超级搜索引擎+超级数据库**

 > 如同原来的百度，你可以在网页上检索任何你想要知道的信息；但是你不能指望他能直接来给你搞定

#### ✅ AI擅长的事情

| 类别       | 具体任务             |
| -------- | ---------------- |
| **文字创作** | 写邮件、写文案、写报告、写总结  |
| **内容处理** | 摘要提炼、翻译、改写、润色    |
| **信息整理** | 归纳要点、提取关键信息、制作表格 |
| **头脑风暴** | 提供创意、分析问题、给出建议   |
| **学习辅助** | 解释概念、回答问题、提供示例   |

#### ❌ AI不擅长/不能做的事情

| 类别         | 说明                       |
| ---------- | ------------------------ |
| **实时信息**   | 不知道训练截止日期之后发生的事情（除非联网搜索） |
| **物理世界操作** | 不能替你打电话、发快递、签合同          |
| **主观判断**   | 不能替你做人生重大决策              |
| **100%准确** | 专业领域（法律、医疗、金融）需要人工核实     |
| **读取人心**   | 不能猜到你没有说出来的需求            |

> ⚠️ **重要原则**: 把AI当作"初稿生成器"和"思路启发者"，而不是"最终答案提供者"。

现在AI可以轻松撰写出漂亮的新年辞，但是想要实现群发则需要后面的各种工具/方案。
之后的章节中将会和大家一起，通过这些方式**打破部分AI的局限**

## 二、主流AI工具入门

### 2.1 概念区分
这里我将我们日常会使用的AI工具分为三个类别：**AI平台、AI模型、垂类AI应用**

#### 2.1.1 AI平台
如我们熟悉的`豆包`、腾讯`元宝`、`通义千问.`....
这些我们可以直接在网页上访问 或者 可以直接在应用商店下载到APP的都属于这一类。


![](../assets/Pasted20260220174024.png)

我们可以直接在其中进行`提问`等AI使用。**这是我们普通人接触AI最多的一个方式。**

同时我们可以发现：我们可以在A平台使用DeepSeek模型、也可以在B平台使用到DeepSeek模型

因为这实际上是厂商基于AI模型进行封装之后的产物，我们普通用户就可以直接像聊天一样进行内容交互（_本质上算一种Agent_）

这类工具通常是依托自己厂商的大模型进行开发的平台，目的是让更多人能够体验到大模型，是一种用户心智的培养（_这就是为什么国产的许多AI平台普通用户使用是免费的，主要是出于推广的需要_）

#### 2.1.2 AI模型

这就是我们最触达AI本质的部分：不管是在大厂封装好的程序中、还是通过第三方软件或者API的方式调用，都是使用的AI模型。

想要体验不同厂商的新模型，则需要去对应的厂商开放平台进行使用。通常一个厂商会有多个不同参数、类型的模型，具体使用什么模型，我们可以根据自己的业务情况，结合官方的说明文档来进行选择。

当然如果我们自己使用的情况下，建议使用最新、最先进的模型——成本上的差异不会太大，但是先进模型将会帮我们节省很多调试时间。
![](../assets/Pasted%20image%2020260220203636.png)

_比如在阿里的平台就能调用所有千问模型及部分第三份热门模型_
但是建议想要用什么模型，找到对应的官网使用是性价比最高的解决方案

总的来说，使用AI模型相对使用AI平台来说难度上略高（后文中会有具体步骤，3分钟就能上手），但是这才能释放出AI的最大能力。

#### 2.1.3 垂类AI应用
上面两种其实主要针对的是文本、代码、图片和视频的AI生成，这些AI模型也是主流AI厂商发力的方向。但是也有部分厂商他们有自己独特的技术优势和行业壁垒，比如AI生成三维模型、设计等领域。

这些通常是有专门的平台（专门用来生成三维模型的Tripo AI）或者集成在原本的软件中（如Adobe软件中的AI功能、figma中make自动设计）

这些AI应用相对来说受众更窄，同时解决特定问题的能力更强，如果市面上的AI无法达到你心仪的效果其实不妨回头看看你专业软件中的AI功能，或许他们将会给你带来不同的惊喜~

### 2.4 如何选择适合自己的AI工具

#### 选择建议

```
如果你是……                    推荐尝试……
─────────────────────────────────────────
完全新手，想快速上手      →  豆包、通义千问
需要处理长文档            →  Kimi、Gemini
需要综合能力              →  ChatGPT、通义千问
需要编程辅助              →  Claude、Gemini、GLM
主要处理中文内容          →  通义千问、豆包、Kimi
```

#### 实用建议

1. **不要贪多**: 先精通1-2个工具，比浅尝辄止10个更有用
2. **免费优先**: 大多数工具提供免费额度，先用免费版熟悉
3. **场景匹配**: 根据你最常用的场景选择工具

### 2.5 为什么要付费

大多数AI工具都提供免费版和付费版，区别主要在：

| 对比项  | 免费版   | 付费版    |
| ---- | ----- | ------ |
| 模型能力 | 基础模型  | 最强模型   |
| 使用次数 | 有限制   | 更高或无限制 |
| 响应速度 | 可能较慢  | 优先响应   |
| 高级功能 | 部分不可用 | 全部功能   |

**付费建议**：
- 🆓 新手阶段：先用免费版熟悉
- 💰 高频用户：如果每天使用，付费版能显著提升效率
- 🎯 特定需求：如果需要最强能力（如复杂推理、编程），考虑付费

## 三、理解AI的局限性

### 3.1 幻觉问题：AI会"一本正经胡说八道"

**什么是AI幻觉？**

AI有时会生成看起来非常可信，但实际上完全错误的信息。这就是"幻觉"（Hallucination）。

#### 常见的幻觉表现

| 类型       | 例子                            |
| -------- | ----------------------------- |
| **编造事实** | "爱因斯坦在1965年获得了诺贝尔奖"（实际是1921年） |
| **虚构引用** | 给出不存在的论文、书籍、网址                |
| **错误归因** | 把A说的话安到B头上                    |
| **逻辑漏洞** | 推理过程看似合理，结论却错误                |
#### 如何应对幻觉？

1. **保持怀疑**: 对AI生成的重要信息进行核实
2. **交叉验证**: 用多个来源验证关键事实
3. **明确告知**: 在提示词中要求"如果不确定，请直接说不知道"

> 💡 **记住**: AI生成的内容是"初稿"，不是"定稿"。重要信息务必核实！

### 3.2 知识时效性

**AI的知识有"截止日期"**。

AI模型是基于训练数据构建的，而训练数据有时间截止点。这意味着：

| 情况             | AI的表现        |
| -------------- | ------------ |
| 训练截止日期之前的事件    | 通常能准确回答      |
| 训练截止日期之后的事件    | 可能完全不知道或信息过时 |
| 持续变化的信息（股价、天气） | 无法提供实时数据     |

#### 解决方案

- 使用**联网搜索**功能（部分工具支持，通常需要单独计费）
- 对于时效性强的信息，通过其他渠道核实

### 3.3 联网检索的准确性问题

很多AI工具支持联网搜索，但要注意：

#### 联网检索的局限

| 问题         | 说明             |
| ---------- | -------------- |
| **信息质量参差** | 搜索结果可能包含不可靠来源  |
| **理解偏差**   | AI可能误解网页内容     |
| **整合错误**   | 多个来源的信息可能被错误整合 |

#### 使用建议

1. 🔍 让AI搜索时，要求它**标注信息来源**
2. 📰 对重要信息，**点击来源链接**自行核实
3. ⚠️ 对于专业领域信息，**谨慎对待**搜索结果

## 四、本章小结

### 核心要点回顾

1. **AI是什么**: 一个读过海量文字、擅长处理的超级助手，但不是万能神
2. **AI能做什么**: 文字创作、内容处理、信息整理、头脑风暴、学习辅助
3. **AI不能做什么**: 实时信息、物理操作、主观决策、100%准确
4. **如何选工具**: 先精通1-2个，根据场景选择，免费优先
5. **AI的局限**: 会产生幻觉、知识有时效性、联网检索需核实
