# 常用技术栈

> **模块目标**: 了解主流编程语言的特点，能够根据项目需求选择合适的技术栈

---

## 一、什么是技术栈

**技术栈（Tech Stack）** 是指开发一个应用所使用的全部技术的集合，包括编程语言、框架、数据库、工具等。

```
一个典型的Web应用技术栈：

前端层：HTML + CSS + JavaScript + Vue/React
   ↓
后端层：Python/Go/Node.js + FastAPI/Gin
   ↓
数据层：MySQL/PostgreSQL + Redis
   ↓
部署层：Docker + Nginx + Linux
```

### 为什么技术栈选择很重要

| 影响 | 说明 |
|-----|------|
| **开发效率** | 选对技术栈，事半功倍 |
| **学习成本** | 不同语言学习难度差异大 |
| **生态支持** | 丰富的库和工具能省很多事 |
| **性能表现** | 不同语言适合不同场景 |
| **维护成本** | 热门技术招人容易，冷门技术维护难 |

## 二、Python

### 2.1 语言特点

Python是一门**简洁易学、生态丰富**的编程语言，是AI时代最热门的语言之一。

```python
# Python代码示例：简洁明了
def greet(name):
    return f"你好，{name}！"

print(greet("AI Navigator"))
```

### 2.2 优点

| 优点 | 说明 |
|-----|------|
| **语法简洁** | 接近自然语言，易于阅读和编写 |
| **AI生态最强** | PyTorch、TensorFlow、LangChain等都在Python |
| **学习资源丰富** | 教程、文档、社区支持完善 |
| **开发效率高** | 代码量少，快速实现想法 |
| **胶水语言** | 能轻松调用C/C++等高性能库 |

### 2.3 缺点

| 缺点 | 说明 | 应对策略 |
|-----|------|---------|
| **运行速度慢** | 解释型语言，性能不如编译型语言 | 用C扩展、异步编程优化 |
| **打包复杂** | 依赖管理、打包分发相对麻烦 | 用Poetry、PyInstaller等工具 |
| **移动端弱** | 不适合开发手机APP | 专注后端和桌面工具 |
| **并发限制** | GIL锁限制多线程性能 | 用多进程、异步IO |

### 2.4 适用场景

| 场景 | 推荐指数 | 说明 |
|-----|---------|------|
| **AI/ML应用** | ⭐⭐⭐⭐⭐ | 最佳选择，生态最完善 |
| **后端服务** | ⭐⭐⭐⭐ | FastAPI/Django很强大 |
| **数据处理** | ⭐⭐⭐⭐⭐ | pandas、numpy是标准工具 |
| **自动化脚本** | ⭐⭐⭐⭐⭐ | 简单直接，效率高 |
| **桌面工具** | ⭐⭐⭐ | PyQt、Tkinter可用 |
| **移动APP** | ⭐ | 不推荐 |

### 2.5 快速体验

```python
# 安装：从 python.org 下载安装

# 一个简单的Web服务（需要先安装：pip install fastapi uvicorn）
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def hello():
    return {"message": "你好，世界！"}

# 运行：uvicorn main:app --reload
```

## 三、JavaScript / TypeScript

### 3.1 语言特点

JavaScript是**Web开发的核心语言**，TypeScript是JavaScript的超集，增加了类型系统。

```javascript
// JavaScript代码示例
function greet(name) {
    return `你好，${name}！`;
}

console.log(greet("AI Navigator"));
```

```typescript
// TypeScript代码示例（增加了类型）
function greet(name: string): string {
    return `你好，${name}！`;
}

console.log(greet("AI Navigator"));
```

### 3.2 优点

| 优点 | 说明 |
|-----|------|
| **前后端通用** | 一门语言搞定全栈开发 |
| **生态最大** | npm包管理器有最多的开源库 |
| **浏览器原生** | 前端只能用JS/TS，无法替代 |
| **异步能力强** | 天然适合IO密集型应用 |
| **TypeScript增强** | 类型系统让大型项目更可维护 |

### 3.3 缺点

| 缺点 | 说明 | 应对策略 |
|-----|------|---------|
| **类型混乱** | JS是弱类型，容易出bug | 使用TypeScript |
| **回调地狱** | 异步代码容易嵌套过深 | 用async/await |
| **生态碎片化** | 工具框架太多，选择困难 | 选主流稳定的 |
| **计算性能一般** | 不适合CPU密集型任务 | 用WebAssembly或后端处理 |

### 3.4 适用场景

| 场景 | 推荐指数 | 说明 |
|-----|---------|------|
| **Web前端** | ⭐⭐⭐⭐⭐ | 唯一选择 |
| **Web后端** | ⭐⭐⭐⭐ | Node.js生态成熟 |
| **桌面应用** | ⭐⭐⭐⭐ | Electron很流行 |
| **移动APP** | ⭐⭐⭐⭐ | React Native跨平台 |
| **AI应用** | ⭐⭐⭐ | 有TensorFlow.js，但不如Python |

### 3.5 快速体验

```javascript
// Node.js后端示例
const http = require('http');

const server = http.createServer((req, res) => {
    res.writeHead(200, {'Content-Type': 'application/json'});
    res.end(JSON.stringify({message: '你好，世界！'}));
});

server.listen(3000, () => {
    console.log('服务器运行在 http://localhost:3000');
});
```

## 四、Golang（Go）

### 4.1 语言特点

Go是Google开发的**高性能、高并发**编程语言，专为云原生时代设计。

```go
// Go代码示例
package main

import "fmt"

func greet(name string) string {
    return fmt.Sprintf("你好，%s！", name)
}

func main() {
    fmt.Println(greet("AI Navigator"))
}
```

### 4.2 优点

| 优点 | 说明 |
|-----|------|
| **性能强劲** | 编译型语言，接近C的性能 |
| **并发原生支持** | goroutine让并发编程变得简单 |
| **部署简单** | 编译成单一二进制文件，无依赖 |
| **语法简洁** | 关键字少，易于学习 |
| **跨平台编译** | 一键编译Windows/Mac/Linux版本 |

### 4.3 缺点

| 缺点 | 说明 | 应对策略 |
|-----|------|---------|
| **AI生态弱** | 机器学习库不如Python丰富 | 通过API调用Python服务 |
| **错误处理繁琐** | 大量if err != nil | 使用错误处理库 |
| **没有泛型**（老版本） | Go 1.18后已支持 | 升级到新版本 |
| **GUI生态弱** | 桌面应用开发不成熟 | 用Wails结合前端 |

### 4.4 适用场景

| 场景 | 推荐指数 | 说明 |
|-----|---------|------|
| **后端服务** | ⭐⭐⭐⭐⭐ | 高性能、高并发首选 |
| **微服务** | ⭐⭐⭐⭐⭐ | 云原生标准语言 |
| **CLI工具** | ⭐⭐⭐⭐⭐ | 编译快、分发简单 |
| **桌面应用** | ⭐⭐⭐⭐ | Wails框架很优秀 |
| **AI应用** | ⭐⭐⭐ | 适合做API层，核心用Python |
| **Web前端** | ⭐ | 可以编译到WASM，但不主流 |

### 4.5 快速体验

```go
// 一个简单的Web服务
package main

import (
    "encoding/json"
    "net/http"
)

func main() {
    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        w.Header().Set("Content-Type", "application/json")
        json.NewEncoder(w).Encode(map[string]string{
            "message": "你好，世界！",
        })
    })
    
    http.ListenAndServe(":8080", nil)
}

// 编译运行：go run main.go
```

## 五、技术栈对比总结

### 5.1 核心指标对比

| 指标 | Python | JavaScript/TS | Go |
|-----|--------|--------------|-----|
| **学习难度** | ⭐ 简单 | ⭐⭐ 中等 | ⭐⭐ 中等 |
| **开发效率** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| **运行性能** | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **AI生态** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ |
| **Web前端** | ❌ | ⭐⭐⭐⭐⭐ | ❌ |
| **Web后端** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **部署难度** | ⭐⭐⭐ | ⭐⭐⭐ | ⭐ |

### 5.2 选择建议

```
如果你是……                              推荐选择……
─────────────────────────────────────────────────────
完全零基础，想快速上手              →  Python
想做AI相关应用                      →  Python
想全栈开发（前后端通吃）            →  JavaScript/TypeScript
追求高性能后端服务                  →  Go
想开发桌面应用                      →  Go + Wails 或 JS + Electron
想开发CLI工具                       →  Go
```

### 5.3 组合使用

实际项目中，往往组合使用多种语言：

```
【方案一：Python全栈】
前端：Vue/React（JS/TS）
后端：Python + FastAPI
适合：AI工具、快速原型

【方案二：Go后端】
前端：Vue/React（JS/TS）
后端：Go + Gin
适合：高性能服务、微服务

【方案三：Node.js全栈】
前端：React/Vue
后端：Node.js + Express/Nest.js
适合：全JavaScript团队
```

## 六、学习建议

### 6.1 初学者路径

```
第一步：选择一门主语言深入学习
    ↓
第二步：掌握基础语法和核心概念
    ↓
第三步：学习相关框架和工具
    ↓
第四步：做实际项目巩固
    ↓
第五步：根据需要学习其他语言
```

### 6.2 不要贪多

> **重要原则**：精通一门语言 > 浅尝十门语言

初学者建议：
1. 先选定一门语言深入学习
2. 能独立完成小项目后再考虑扩展
3. 语言只是工具，解决问题的能力才是核心

### 6.3 利用AI学习编程

现在学习编程比以前容易得多：

```
你：请解释这段Python代码是什么意思
AI：这段代码的作用是……

你：请帮我写一个读取Excel文件的Python代码
AI：好的，这是示例代码……

你：这段代码报错了，请帮我看看问题在哪
AI：问题出在第5行，应该改成……
```

## 七、本章小结

### 核心要点回顾

1. **Python**：简洁易学，AI生态最强，适合快速开发和AI应用
2. **JavaScript/TypeScript**：前后端通用，生态最大，Web开发必备
3. **Go**：高性能、部署简单，适合后端服务和CLI工具
4. **选择原则**：根据项目需求和自身情况选择，不必贪多
