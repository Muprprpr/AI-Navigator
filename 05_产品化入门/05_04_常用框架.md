# 常用框架

> **模块目标**: 理解框架的价值，了解主流框架的特点，能够选择合适的框架快速搭建应用

---

## 一、为什么要用框架

### 1.1 没有框架的日子

假设你要开发一个Web应用，从零开始需要处理：

```
❌ 不使用框架：
- 手动处理HTTP请求解析
- 手动管理路由（哪个URL对应哪个功能）
- 手动处理数据库连接
- 手动实现用户认证
- 手动处理安全防护（XSS、CSRF等）
- 手动组织代码结构
- ……
```

### 1.2 框架解决的问题

**框架（Framework）** 是一套预先设计好的代码结构和工具集，帮你处理通用问题，让你专注于业务逻辑。

```
✅ 使用框架：
- 框架自动处理HTTP请求
- 框架提供路由系统
- 框架提供数据库ORM
- 框架提供认证模块
- 框架内置安全防护
- 框架规范代码结构
- 你只需要写业务代码
```

### 1.3 框架的核心价值

| 价值 | 说明 |
|-----|------|
| **提高效率** | 轮子不用重复造，开箱即用 |
| **规范代码** | 统一的代码结构，团队协作更容易 |
| **最佳实践** | 框架集成了行业最佳实践 |
| **生态支持** | 丰富的插件和社区支持 |
| **降低门槛** | 不需要从底层学起 |

## 二、框架脚手架（CLI）

### 2.1 什么是脚手架

**脚手架（Scaffold/CLI）** 是框架提供的命令行工具，帮你快速创建项目骨架。

```
# 没有脚手架
1. 手动创建文件夹
2. 手动创建各种配置文件
3. 手动安装依赖
4. 手动写基础代码
……（可能需要1小时）

# 有脚手架
npm create vue@latest my-app
……（1分钟搞定）
```

### 2.2 常见脚手架命令

| 框架 | 脚手架命令 |
|-----|-----------|
| **Vue** | `npm create vue@latest` |
| **React** | `npx create-react-app` 或 `npm create vite@latest` |
| **FastAPI** | `pip install fastapi` |
| **Django** | `django-admin startproject` |
| **Go/Gin** | 无官方脚手架，手动创建 |
| **Wails** | `wails init -n myapp` |

### 2.3 脚手架创建的项目结构

以Vue项目为例：

```
my-vue-app/
├── node_modules/     # 依赖包（自动生成）
├── public/           # 静态资源
├── src/              # 源代码
│   ├── assets/       # 资源文件
│   ├── components/   # 组件
│   ├── views/        # 页面
│   ├── App.vue       # 根组件
│   └── main.js       # 入口文件
├── index.html        # HTML模板
├── package.json      # 项目配置
└── vite.config.js    # 构建配置
```

## 三、前端框架

### 3.1 Vue

**Vue** 是一款渐进式JavaScript框架，以易学易用著称。

#### 特点

| 特点 | 说明 |
|-----|------|
| **学习曲线平缓** | 概念简单，上手快 |
| **中文生态好** | 尤雨溪（华人）创建，中文文档完善 |
| **模板语法直观** | HTML模板 + 指令，容易理解 |
| **双向绑定** | 数据和视图自动同步 |
| **组件化** | 可复用的UI组件 |

#### 适用场景

- ✅ 初学者入门前端框架
- ✅ 中小型Web应用
- ✅ 企业内部管理系统
- ✅ 需要快速开发的项目

#### 快速体验

```vue
<!-- Vue单文件组件 -->
<template>
  <div>
    <h1>{{ message }}</h1>
    <button @click="count++">点击次数: {{ count }}</button>
  </div>
</template>

<script>
export default {
  data() {
    return {
      message: '你好，Vue！',
      count: 0
    }
  }
}
</script>

<style scoped>
h1 { color: #42b983; }
</style>
```

---

### 3.2 React

**React** 是Facebook开发的UI库，是最流行的前端框架之一。

#### 特点

| 特点 | 说明 |
|-----|------|
| **生态最大** | 组件库、工具、社区最丰富 |
| **JSX语法** | JavaScript中写HTML |
| **虚拟DOM** | 高效的页面更新机制 |
| **灵活自由** | 只负责视图层，其他自由选择 |
| **大厂背书** | Facebook、Netflix等大厂使用 |

#### 适用场景

- ✅ 大型复杂Web应用
- ✅ 需要丰富生态支持的项目
- ✅ 移动端开发（React Native）
- ✅ 有一定前端基础的开发者

#### 快速体验

```jsx
// React组件
import { useState } from 'react';

function App() {
  const [count, setCount] = useState(0);
  
  return (
    <div>
      <h1>你好，React！</h1>
      <button onClick={() => setCount(count + 1)}>
        点击次数: {count}
      </button>
    </div>
  );
}

export default App;
```

---

### 3.3 Vue vs React 选择建议

| 对比项 | Vue | React |
|-------|-----|-------|
| **学习难度** | ⭐ 简单 | ⭐⭐ 中等 |
| **生态规模** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **中文资源** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| **灵活性** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **适合初学者** | ✅ 是 | ⚠️ 有一定门槛 |

**选择建议**：
- 初学者 → **Vue**（上手快，中文资源多）
- 追求生态 → **React**（组件库最多）
- 团队已有经验 → 跟随团队选择

## 四、后端框架

### 4.1 Python后端框架

#### FastAPI

**FastAPI** 是现代、快速的Python Web框架，特别适合构建API。

| 特点 | 说明 |
|-----|------|
| **性能极高** | 接近Go/Node.js的性能 |
| **自动文档** | 自动生成Swagger API文档 |
| **类型提示** | 基于Python类型注解 |
| **异步支持** | 原生支持async/await |
| **开发效率高** | 代码简洁，开发快 |

```python
# FastAPI示例
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def hello():
    return {"message": "你好，FastAPI！"}

@app.get("/users/{user_id}")
def get_user(user_id: int):
    return {"user_id": user_id}
```

**适合场景**：AI应用后端、微服务、RESTful API

---

#### Django

**Django** 是全功能的Python Web框架，"开箱即用"。

| 特点 | 说明 |
|-----|------|
| **功能全面** | Admin后台、ORM、认证、缓存都有 |
| **安全可靠** | 内置多种安全防护 |
| **成熟稳定** | 十多年历史，文档完善 |
| **适合大型项目** | 很多知名网站使用 |

```python
# Django示例
from django.http import JsonResponse

def hello(request):
    return JsonResponse({"message": "你好，Django！"})
```

**适合场景**：内容管理系统、电商网站、企业级应用

---

#### FastAPI vs Django

| 对比项 | FastAPI | Django |
|-------|---------|--------|
| **性能** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| **学习曲线** | ⭐⭐ | ⭐⭐⭐ |
| **功能完整度** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **API开发** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| **全栈开发** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

**选择建议**：
- 构建API服务 → **FastAPI**
- 快速开发完整网站 → **Django**
- AI应用后端 → **FastAPI**（异步支持好）

---

### 4.2 Go后端框架

#### Gin

**Gin** 是Go语言最流行的Web框架，轻量高性能。

| 特点 | 说明 |
|-----|------|
| **极高性能** | 性能是Go框架中最高的之一 |
| **轻量简洁** | 核心功能精简，扩展灵活 |
| **中间件丰富** | 有大量第三方中间件 |
| **API友好** | 专门为API设计 |

```go
// Gin示例
package main

import "github.com/gin-gonic/gin"

func main() {
    r := gin.Default()
    
    r.GET("/", func(c *gin.Context) {
        c.JSON(200, gin.H{
            "message": "你好，Gin！",
        })
    })
    
    r.Run(":8080")
}
```

---

#### GoFrame (GF)

**GoFrame** 是功能更全面的Go框架，提供企业级开发支持。

| 特点 | 说明 |
|-----|------|
| **功能全面** | ORM、缓存、配置、日志等都有 |
| **中文文档** | 国产框架，中文文档完善 |
| **企业级** | 适合大型项目 |
| **工具链** | 提供代码生成等工具 |

**选择建议**：
- 追求极致性能 → **Gin**
- 需要完整功能 → **GoFrame**
- 国人团队/中文支持 → **GoFrame**

## 五、全栈框架

### 5.1 什么是全栈框架

**全栈框架** 让你用一套技术栈同时开发前端和后端。

### 5.2 Wails（Go + 前端）

**Wails** 是用Go开发桌面应用的框架，前端可以使用Vue/React/Svelte等。

| 特点 | 说明 |
|-----|------|
| **Go后端** | 性能强，打包简单 |
| **前端自由** | 支持Vue/React/Svelte等 |
| **原生体验** | 使用系统原生渲染 |
| **打包体积小** | 比Electron小很多 |
| **跨平台** | Windows/Mac/Linux |

```
Wails项目结构：
my-app/
├── build/          # 构建配置
├── frontend/       # 前端代码（Vue/React等）
├── app.go          # Go后端代码
├── main.go         # 入口文件
└── wails.json      # Wails配置
```

**适合场景**：桌面工具、本地AI应用

---

### 5.3 其他全栈方案

| 框架 | 技术栈 | 特点 |
|-----|-------|------|
| **Electron** | JS + Chromium | 生态成熟，但体积大 |
| **Tauri** | Rust + 前端 | 体积小，但Rust学习成本高 |
| **Next.js** | React全栈 | SSR支持，适合Web应用 |
| **Nuxt.js** | Vue全栈 | SSR支持，适合Web应用 |

## 六、框架选择指南

### 6.1 按项目类型选择

```
【Web应用】
前端：Vue 或 React
后端：FastAPI（Python）或 Gin（Go）

【桌面应用】
首选：Wails（Go + Vue/React）
备选：Electron（JS全栈）

【移动应用】
跨平台：React Native 或 Flutter

【API服务】
Python：FastAPI
Go：Gin

【快速原型】
Python：Streamlit（数据应用）
Python：Gradio（AI应用）
```

### 6.2 按个人情况选择

```
【完全零基础】
推荐：Vue + FastAPI
理由：学习曲线平缓，中文资源多

【有JS基础】
推荐：React + Node.js
理由：技能复用，全JS开发

【追求性能】
推荐：Vue/React + Gin
理由：Go后端性能强

【AI应用】
推荐：Vue/React + FastAPI
理由：Python AI生态最好
```
