# 前端设计

> **模块目标**: 学会使用设计工具绘制界面原型，掌握从设计稿到代码的转换方法

---

## 一、为什么需要前端设计

### 1.1 设计的价值

在写代码之前先做设计，有这些好处：

| 好处         | 说明           |
| ---------- | ------------ |
| **想清楚再动手** | 避免边写边改，浪费时间  |
| **统一视觉风格** | 整体规划颜色、字体、间距 |
| **方便沟通**   | 让别人直观理解你的想法  |
| **降低返工成本** | 改设计稿比改代码快得多  |

### 1.2 设计的层次

```
低保真原型          →        高保真设计稿        →        实现代码
   ↓                          ↓                       ↓
快速验证想法              确定视觉细节              最终产品
（纸笔/Pencil）          （Figma）               （写代码）
```

## 二、Figma - 专业级设计工具

### 2.1 Figma简介

**Figma** 是目前最流行的UI设计工具，免费且功能强大。

| 特点 | 说明 |
|-----|------|
| **在线协作** | 浏览器直接使用，支持多人协作 |
| **免费够用** | 个人免费版功能足够 |
| **组件系统** | 可复用的UI组件 |
| **自动布局** | 响应式设计支持 |
| **插件丰富** | 大量第三方插件 |

### 2.2 快速上手

#### 访问Figma

1. 打开浏览器，访问 [figma.com](https://www.figma.com)
2. 注册免费账号
3. 点击 "New design file" 创建新文件

#### 基本操作

| 操作 | 快捷键/方法 |
|-----|-----------|
| 选择工具 | V |
| 矩形工具 | R |
| 文字工具 | T |
| 画框工具 | F |
| 缩放画布 | 鼠标滚轮 |
| 移动画布 | 按住空格+拖动 |
| 复制 | Ctrl/Cmd + D |
| 删除 | Delete/Backspace |

### 2.3 设计一个简单页面

#### 第一步：创建画框

```
1. 按 F 选择 Frame 工具
2. 在右侧面板选择预设尺寸（如 Desktop）
3. 或自定义尺寸（如 1440 x 900）
```

#### 第二步：添加基础元素

```
添加矩形（按钮）：
1. 按 R 选择矩形工具
2. 拖动绘制矩形
3. 在右侧面板调整：
   - 填充颜色：#4F46E5（紫色）
   - 圆角：8px
   - 宽高：120 x 40

添加文字：
1. 按 T 选择文字工具
2. 点击画布输入文字
3. 调整字体、大小、颜色
```

#### 第三步：使用自动布局

自动布局让元素自动排列，类似CSS的Flexbox：

```
1. 选中多个元素
2. 右键 → Add auto layout
3. 调整间距和对齐方式
```

### 2.4 Figma实用技巧

| 技巧 | 说明 |
|-----|------|
| **使用组件** | 创建可复用的按钮、卡片等 |
| **使用样式** | 统一颜色、字体、效果 |
| **使用网格** | 对齐元素更整齐 |
| **善用插件** | Iconify、Unsplash等插件提高效率 |

## 三、Pencil - 轻量级原型工具

### 3.1 Pencil简介

**Pencil** 是一款免费开源的原型设计工具，适合快速绘制低保真原型。

| 特点 | 说明 |
|-----|------|
| **完全免费** | 开源软件，无任何费用 |
| **离线使用** | 桌面应用，无需联网 |
| **简单易用** | 拖拽式操作，上手快 |
| **模板丰富** | 内置多种UI组件库 |

### 3.2 安装Pencil

```
1. 访问 GitHub: github.com/evolus/pencil
2. 下载对应系统的安装包
3. 安装完成后打开
```

### 3.3 使用Pencil绘制原型

#### 创建新文档

```
1. File → New Document
2. 选择页面尺寸（如 Web 1280x800）
```

#### 添加组件

```
1. 左侧面板选择组件库
2. 拖拽组件到画布
3. 调整大小和位置
```

#### 常用组件

| 组件 | 用途 |
|-----|------|
| **Rectangle** | 容器、按钮 |
| **Text** | 文字标签 |
| **Text Editor** | 输入框 |
| **Button** | 按钮 |
| **Drop down** | 下拉选择 |

### 3.4 Pencil vs Figma

| 对比项 | Figma | Pencil |
|-------|-------|--------|
| **使用方式** | 在线 | 离线 |
| **功能丰富度** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| **学习难度** | ⭐⭐⭐ | ⭐ |
| **协作能力** | ⭐⭐⭐⭐⭐ | ❌ |
| **适合场景** | 高保真设计 | 快速原型 |

**选择建议**：
- 需要精美设计 → **Figma**
- 快速画草图 → **Pencil**
- 团队协作 → **Figma**
- 离线使用 → **Pencil**

## 四、UI设计原则

### 4.1 核心设计原则

无论使用什么工具，好的设计都遵循以下原则：

#### 1. 对齐（Alignment）

```
❌ 不对齐：
┌─────────────────────────┐
│  标题                   │
│     内容文字            │
│ 按钮                    │
└─────────────────────────┘

✅ 对齐：
┌─────────────────────────┐
│  标题                   │
│  内容文字               │
│  按钮                   │
└─────────────────────────┘
```

#### 2. 对比（Contrast）

```
通过对比突出重点：
- 大标题 vs 小正文
- 深色文字 vs 浅色背景
- 粗体 vs 细体
```

#### 3. 重复（Repetition）

```
保持一致：
- 相同功能的按钮样式一致
- 页面间保持统一风格
- 间距使用规律
```

#### 4. 亲密性（Proximity）

```
相关的元素靠近：
┌─────────────────────────┐
│  个人信息               │  ← 标题和内容靠近
│  姓名：张三             │
│  电话：138xxxx          │
│                         │  ← 不同组之间有间距
│  工作信息               │
│  公司：XX科技           │
└─────────────────────────┘
```

### 4.2 配色建议

#### 配色原则

| 原则 | 说明 |
|-----|------|
| **主色** | 1-2个，用于品牌识别 |
| **辅助色** | 2-3个，用于点缀 |
| **中性色** | 黑白灰，用于文字和背景 |
| **功能色** | 成功绿、警告黄、错误红 |

#### 常用配色方案

```
【商务蓝】
主色：#2563EB
背景：#F8FAFC
文字：#1E293B

【清新绿】
主色：#10B981
背景：#ECFDF5
文字：#064E3B

【科技紫】
主色：#7C3AED
背景：#F5F3FF
文字：#2E1065
```

### 4.3 字体建议

| 类型 | 中文推荐 | 英文推荐 |
|-----|---------|---------|
| **标题** | 思源黑体、苹方 | Inter、Roboto |
| **正文** | 思源宋体、苹方 | Inter、Open Sans |
| **代码** | 等距更纱黑体 | JetBrains Mono、Fira Code |

## 五、从设计稿到代码

### 5.1 设计稿标注

在Figma中，可以查看设计稿的各种参数：

```
查看方法：
1. 选中元素
2. 右侧面板查看：
   - 尺寸（W × H）
   - 位置（X, Y）
   - 颜色（Fill）
   - 字体（Typography）
   - 间距（Auto layout）
```

### 5.2 手动还原设计稿

#### 示例：将设计稿转为HTML/CSS

**设计稿信息**：
```
按钮：
- 宽度：120px
- 高度：40px
- 背景色：#4F46E5
- 圆角：8px
- 文字：开始生成
- 文字颜色：白色
- 文字大小：14px
- 居中
```

**转换为代码**：

```html
<button class="btn">开始生成</button>

<style>
.btn {
  width: 120px;
  height: 40px;
  background-color: #4F46E5;
  border-radius: 8px;
  color: white;
  font-size: 14px;
  border: none;
  cursor: pointer;
  display: flex;
  align-items: center;
  justify-content: center;
}
</style>
```

### 5.3 使用AI辅助转换

可以用AI帮你把设计稿转换成代码：

```
【提示词示例】

我有一个按钮的设计稿，请帮我转换成HTML和CSS代码：
- 宽度：120px
- 高度：40px
- 背景色：#4F46E5（紫色）
- 圆角：8px
- 文字："开始生成"，白色，14px
- 鼠标悬停时背景变深

请提供完整的HTML和CSS代码。
```

### 5.4 Figma插件推荐

| 插件 | 功能 |
|-----|------|
| **Figma to HTML** | 导出HTML代码 |
| **Builder.io** | 设计稿转代码 |
| **Iconify** | 海量图标库 |
| **Unsplash** | 免费图片 |
| **Color Palette** | 配色方案生成 |

## 七、本章小结

### 核心要点回顾

1. **Figma**：专业级在线设计工具，功能强大，支持协作
2. **Pencil**：轻量级离线原型工具，简单易用
3. **设计原则**：对齐、对比、重复、亲密性
4. **设计到代码**：标注参数，手动或AI辅助转换

### 工具选择建议

| 场景 | 推荐工具 |
|-----|---------|
| 专业设计 | Figma |
| 快速原型 | Pencil |
| 团队协作 | Figma |
| 离线使用 | Pencil |
| 不想设计 | 直接用UI组件库 |
